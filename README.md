## 我适合用这个吗？

✅ 你有一堆本地 mp3，想在 Android 上听，不想被会员订阅或专辑/艺人分类绑架  
✅ 想要"想跑步就放跑步的歌、想冥想就放冥想的歌"这种感觉  
✅ 不介意花 30 秒注册一个**免费的** AI Key（智谱 GLM-4-Flash 永久免费）  
✅ 接受这是一个个人项目，没有 App Store、没有自动更新、没有客服

❌ 你只想用流媒体（这个不联网拉歌，只放你手机里的本地文件）  
❌ 你完全不想配 AI（不配也能用，但场景功能就没意义了，只能按文件夹听歌）

## 快速上手（4 步）

### 1. 下载 APK 装到手机

到 [Releases 页面](https://github.com/marsiot/chouchoumusic/releases) 下载最新的 `chouchou-music-vX.X.X.apk`，在手机上点开安装。

第一次会提示"未知来源应用"——按手机的提示去设置里允许一下就行。

> ⚠️ 这是 **debug 签名**，仅供个人使用，不会上 Play Store。

### 2. 把你的音乐放到手机的 `Music/` 目录

用 USB 数据线连接手机，把音乐文件复制到手机的 **`/Music/`** 目录下，**按"分类"组织成子文件夹**，例如：

```
内部存储/
└── Music/
    ├── 周杰伦/
    │   ├── 晴天.mp3
    │   ├── 七里香.mp3
    │   └── ...
    ├── 赵雷/
    │   ├── 成都.mp3
    │   └── ...
    ├── 跑步专用/        ← 可以按用途分
    │   ├── ...
    └── ...
```

**子文件夹的名字就是 App 里"文件夹"那一列显示的名字**——可以按歌手分、按专辑分、按用途分，随你怎么舒服。

> 如果有几首散文件直接放在 `/Music/` 下面没进任何子目录，它们会出现在一个叫 **"未分类"** 的虚拟文件夹里——可以在 App 内点歌曲的 ⋯ 菜单把它移到正确的文件夹去。

### 3. 第一次打开 App，授权读写文件

打开"臭臭音乐"，会让你授一个**"管理所有文件"**的权限（Android 11+ 需要这个才能读写 Music 目录里的 `.lrc` 歌词文件）——跟着提示同意就好。

### 4. 配置 AI（**强烈建议，2 分钟搞定**）

不配 AI，场景功能就是空的（"跑步"列表里没有歌）。配上 AI 后，它会自动判断每首歌适合哪些场景。

**最省事：用智谱 GLM-4-Flash（完全免费）**

1. 浏览器打开 [智谱开放平台](https://open.bigmodel.cn/usercenter/proj-mgmt/apikeys) → 注册/登录 → 创建一个 API Key → 复制
2. 回到 App → 右上角 **设置** → 找到 **智谱 GLM-4-Flash** → 开启 → 粘贴 Key → 保存
3. 退出设置，App 自动开始扫描所有歌——上方会有进度条 `AI 分类中 12/50`

扫完后进任何一个场景（跑步、冥想），就会看到适合的歌已经在里面了。

> 担心隐私？API Key 只存在手机本地，AI 只会收到歌曲的**歌名 + 文件夹名**（用作歌手提示），不会上传音频。
>
> 流量：每 10 首歌一次 API 调用，整库 200 首大约 20 次调用，几 KB 流量。

## 怎么用

### 主界面看到的东西

```
┌─────────────────────────────┐
│  臭臭音乐               ⚙   │  ← 顶部：App 名 + 设置
├─────────────────────────────┤
│  全部                       │
│  2 个场景 · 5 个文件夹 · 87 首 │  ← 副标题：库容量
├─────────────────────────────┤
│  跑步                  12 > │  ← 场景（前面）
│  冥想                   8 > │
│  + 添加场景                 │
│  周杰伦                25 > │  ← 文件夹（后面）
│  赵雷                  10 > │
│  ...                        │
│  未分类                 3 > │  ← 散文件总在最后
├─────────────────────────────┤
│  ▮▮▮ 正在播放: xxx          │  ← 迷你播放器
│  ⏮ ⏯ ⏭                       │
└─────────────────────────────┘
```

### 添加自己的场景

点 **+ 添加场景** → 输入场景名（如"夜跑"、"做饭"、"写代码"）→ **选歌曲数量**（默认 10 首，可拨到"不限"）→ 确认。

AI 立即开始判断哪些歌适合，**够目标数量就停**（省 token）。

### 换一批新的歌

某个场景听腻了？进场景 → 右上角 ⋯ → **更换曲目** → AI 从候选池里挑一批不一样的进来。

### 手工调整标签

某首歌 AI 判断错了？或者你想强制把它加进某场景？

长按音乐列表里某首歌的 **⋯** 按钮（每行右边）→ 顶上有"场景标签"那一栏 → 直接点勾选/取消任何场景 → 立即生效。

**手动改过的歌**会被锁定，以后 AI 重扫不会再改它的标签。

### 歌词

播放时迷你播放器会显示一行实时歌词。点一下迷你播放器进**歌词详情页**，可以看完整歌词同步滚动。

- 第一次播放某首歌时，App 自动从网易云抓 LRC 歌词
- 抓错了？歌词页右上角 🔍 按钮 → 手工输入正确的歌名/歌手 → 重新搜
- 没有歌词的歌也能进详情页（不会再误退到桌面）

### 文件管理

每首歌的 ⋯ 菜单里可以：
- **重命名** —— 改文件名
- **移动到...** —— 把歌挪到另一个文件夹
- **复制到...** —— 复制一份到另一个文件夹（标签也跟着复制）
- **删除** —— 文件 + 它的 `.lrc` 歌词一起删

文件夹和场景的 ⋯ 菜单里也有重命名、删除。**重命名场景**会自动让 AI 用新名字重新判断一遍（因为"跑步"和"慢跑"AI 的理解可能不一样）。

### 后台播放

按手机的"返回键"不会停音乐——App 会进入后台继续播放，状态栏有通知告诉你当前在放什么。

要彻底停掉，从"最近任务"里把它划走就行。

### 拖拽改顺序

长按音乐列表里任意一首歌可以**上下拖动**调整顺序，文件夹内和场景内都行，顺序会被记下来。

## 常见问题

**Q：扫描了一会儿就停了怎么办？**  
A：检查右上角 ⚙ 设置里你启用的 AI 服务商是否还有余额/可用。AI 调用失败时会弹一个 toast 告诉你具体错误。**智谱的免费额度极其充足**，建议至少把它启用作为兜底。

**Q：AI 给某些歌打的标签我不喜欢，怎么办？**  
A：直接长按或点那首歌的 ⋯ → 手动改场景标签。**改过的歌 AI 永远不会再动它**。

**Q：可以挂多家 AI 吗？**  
A：可以，设置里把多家都开启 → App 会自动按顺序故障转移（一家失败就用下一家）。

**Q：换了手机怎么迁移？**  
A：把 `/Music/` 整个目录拷到新手机相同位置即可——歌曲的场景标签和歌词都保存在每首歌旁边的同名 `.lrc` 文件里。**API Key、自定义场景这些设置需要重新配置一次**。

**Q：是否会扫描手机里其它目录的音乐？**  
A：默认只扫 `/Music/`。如果你的音乐放在其它目录（如 `Download/Music/` 或 `KuwoMusic/`），可以在设置里改"扫描目录"。

**Q：占用流量大吗？**  
A：AI 分类几乎可以忽略不计（一次几 KB）。歌词匹配从网易云下 LRC，每首歌 1-2 KB。**两者都只在首次处理时调用一次，之后不再重复请求。**

**Q：会不会上传我的歌曲？**  
A：不会。AI 只会收到歌名和文件夹名（用作"歌手"线索），不传任何音频内容。歌词只是从网易云**下载**公开歌词。

## 想自己改

代码在 GitHub: https://github.com/marsiot/chouchoumusic

技术栈：Java 8 + AppCompat + Material + RecyclerView + 原生 MediaPlayer，没有用 Kotlin/Compose/ExoPlayer。逻辑集中在 `MainActivity.java`，没有过度抽象。

```bash
# 一键编 + 装到连接的手机
./build_and_run.bat
```

更多脚本说明见 [SCRIPTS.md](SCRIPTS.md)。

歌曲的场景标签和歌词都存在每首歌旁边的同名 `.lrc` 里，开头一行特殊元数据：

```
[chouchou:scenes=跑步,健身;classified=1;manual=1;lyrics=1]
[ti:歌名]
[ar:歌手]
[00:12.34]这是第一句歌词
```
